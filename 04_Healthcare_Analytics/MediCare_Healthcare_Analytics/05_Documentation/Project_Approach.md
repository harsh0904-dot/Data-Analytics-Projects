# Project Approach

## 1) Data Collection

- Use raw hospital patient CSV as the source file.
- Load the dataset into Python for cleaning and validation.

## 2) Data Preparation

- Standardize column names and text values.
- Parse dates and validate numeric fields.
- Correct inconsistent state names and remove duplicates.
- Save cleaned output to Excel for reporting use.

## 3) Analysis Layer

- Build KPI SQL queries for executive metrics.
- Build analysis queries for condition, state, outcome, and insurance trends.

## 4) Dashboard Layer

- Design three Power BI pages for executive, clinical, and financial analysis.
- Use cards, charts, tables, waterfall, and decomposition visuals.

## 5) Validation

- Cross-check totals between Python, SQL, and Power BI.
- Validate screenshot metrics before final documentation.
